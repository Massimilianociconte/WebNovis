# Live Query Briefs — 13 aprile 2026

Questo file raccoglie gli angoli di ricerca e di posizionamento usati per i rewrite manuali delle pagine `agenzia-web-*`.

Obiettivo:
- evitare riscritture generiche
- ancorare il copy a query commerciali reali o realistiche per SERP locali italiane
- differenziare le pagine per intent, landmark e contesto economico

## Metodo

Le pagine sono state riscritte incrociando:
- query locali principali, come `agenzia web [comune]`, `realizzazione siti web [comune]`, `siti web [comune]`
- pattern emersi da SERP e competitor locali o pseudo-locali
- contesto economico del comune già presente in `data/cities.json`
- landmark e nodi territoriali che cambiano davvero l'intento commerciale

## Query Families

Le famiglie di query più utili per il cluster sono:
- `agenzia web [comune]`
- `realizzazione siti web [comune]`
- `siti web [comune]`
- `e-commerce [comune]`
- `seo locale [comune]`
- query settoriali ibride, come `sito web hotel [comune]`, `e-commerce calzature [comune]`, `sito web studio medico [comune]`

## Regole editoriali che stanno funzionando

- Aprire con contesto locale vero, non con una definizione piatta del comune
- Inserire 2-3 query commerciali nel copy in modo naturale, senza keyword stuffing
- Aggiungere un blocco `geo-proof` con micro-casi e scenari d'uso
- Riscrivere le FAQ come obiezioni reali del cliente locale, non come FAQ generiche del template
- Spostare il confronto competitivo da “prezzo basso” a “qualità del progetto + vicinanza + utilità commerciale”
- Evitare claim non verificabili o numeri troppo precisi quando non supportati
